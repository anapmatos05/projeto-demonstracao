# Estudo de Desenvolvimento Colaborativo: Git & GitHub

Este repositório serve como suporte à apresentação sobre a utilização do **Git** e do **GitHub** no desenvolvimento de software em equipa, onde utilizamos as IDEs do **NetBeans**.

### 1. Introdução ao Controlo de Versões
Conforme detalhado no manual *Pro Git*, o Git é um sistema **distribuído**. 
* Cada programador possui uma cópia integral do histórico.
* Permite reverter erros e comparar versões de forma instantânea.

### 2. O Ciclo de Trabalho na IDE (IntelliJ / NetBeans)
A integração do Git nas IDEs permite um feedback visual imediato:
* **Ficheiros Verdes:** Novos ficheiros (untracked).
* **Ficheiros Azuis:** Ficheiros modificados.
* **Ferramenta de Diff:** Comparação lado a lado antes de cada commit.

### 3. Colaboração com GitHub Flow
A colaboração eficiente segue estes passos:
1. **Branching:** Criação de ramos para novas funcionalidades (ex: `feat-login`).
2. **Commit & Push:** Gravação local e envio para o servidor.
3. **Pull Request (PR):** Abertura de uma proposta de integração para revisão de código.
4. **Merge:** União do código aprovado com a branch principal (`main`).

### 4. Resolução de Conflitos
Demonstração da ferramenta visual de **3-Way Merge** da IDE, utilizada quando dois membros da equipa alteram a mesma linha de código simultaneamente.
